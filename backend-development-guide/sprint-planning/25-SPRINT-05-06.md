**Task 1.2.1: Feed Generation Strategy**

- **Assignee:** Backend Dev 1
- **Story Points:** 8
- **Description:**
  - Design feed algorithm
  - Implement FeedGenerator domain service
  - Calculate post relevance score
  - Add pagination support
  - Implement caching strategy

**Feed Algorithm:**

```
Score = (Time_Score * 0.4) +
        (Engagement_Score * 0.3) +
        (Author_Score * 0.2) +
        (Content_Score * 0.1)

Time_Score:
  - Recent posts (< 24h): 100
  - 1-3 days: 75
  - 3-7 days: 50
  - > 7 days: 25

Engagement_Score:
  - Likes: each like = +2 points
  - Comments: each comment = +5 points
  - Cap at 100

Author_Score:
  - Following: 100
  - Same profession: 75
  - Different profession: 50

Content_Score:
  - Has images: +20
  - Length > 200 chars: +10
```

**Feed Query:**

```java
public record FeedQuery(
    UserId userId,
    int page,
    int size,
    Optional<Profession> professionFilter
) {}
```

**Caching Strategy:**

```
Cache Key: feed:{userId}:{page}:{professionFilter}
TTL: 5 minutes
Invalidation: On new post by followed user
```

**Feed Query (Native SQL):**

```sql
WITH user_follows AS (
    SELECT following_id
    FROM follows
    WHERE follower_id = :userId
),
scored_posts AS (
    SELECT
        p.*,
        u.first_name,
        u.last_name,
        u.profile_image_url,
        -- Time score (0-100)
        CASE
            WHEN p.created_at > NOW() - INTERVAL '24 hours' THEN 100
            WHEN p.created_at > NOW() - INTERVAL '3 days' THEN 75
            WHEN p.created_at > NOW() - INTERVAL '7 days' THEN 50
            ELSE 25
        END AS time_score,
        -- Engagement score (0-100)
        LEAST(100, (p.like_count * 2) + (p.comment_count * 5)) AS engagement_score,
        -- Author score (0-100)
        CASE
            WHEN p.author_id IN (SELECT following_id FROM user_follows) THEN 100
            WHEN p.profession = :userProfession THEN 75
            ELSE 50
        END AS author_score,
        -- Content score (0-100)
        (CASE WHEN array_length(p.image_urls, 1) > 0 THEN 20 ELSE 0 END +
         CASE WHEN length(p.content) > 200 THEN 10 ELSE 0 END) AS content_score
    FROM posts p
    JOIN users u ON p.author_id = u.id
    WHERE p.deleted = false
        AND u.active = true
)
SELECT
    *,
    (time_score * 0.4 + engagement_score * 0.3 +
     author_score * 0.2 + content_score * 0.1) AS relevance_score
FROM scored_posts
ORDER BY relevance_score DESC
LIMIT :size OFFSET :offset;
```

## 7. Database Performance

### Query Optimization:

```sql
-- Before: 2.5s (N+1 queries)
SELECT * FROM posts WHERE author_id IN (SELECT following_id FROM follows);

-- After: < 500ms (single query with JOIN)
SELECT p.*, u.first_name, u.last_name
FROM posts p
JOIN follows f ON p.author_id = f.following_id
JOIN users u ON p.author_id = u.id
WHERE f.follower_id = :userId;
```
