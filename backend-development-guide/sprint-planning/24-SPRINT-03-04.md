**Task 1.2.3: AI Verification Processor**

- **Assignee:** Backend Dev 1
- **Story Points:** 8
- **Description:**
  - Create AIVerificationProcessor service
  - Orchestrate Rekognition calls
  - Implement auto-approval logic
  - Route to manual review if needed
  - Store AI verification results

**Processing Flow:**

```
1. Download images from S3
2. Compare faces (Rekognition)
3. Extract document text (OCR)
4. Validate document number match
5. Calculate overall confidence score
6. Auto-approve if confidence > 95%
7. Route to manual review if 85% < confidence < 95%
8. Auto-reject if confidence < 85%
9. Store results and update status
```

**Confidence Calculation:**

```java
float overallConfidence =
    (faceMatchScore * 0.6) +      // 60% weight
    (documentValidScore * 0.3) +   // 30% weight
    (textMatchScore * 0.1);        // 10% weight
```

**Acceptance Criteria:**

- ✅ AI processing completes within 10 seconds
- ✅ Auto-approval works for high confidence
- ✅ Manual review queue populated correctly
- ✅ Results stored in database

## 4. Risks & Mitigation

### Risk 1: AWS Rekognition Cost

**Impact:** High
**Probability:** Medium
**Mitigation:**

- Cache AI results (don't re-process same images)
- Implement request throttling (max 100/hour)
- Use Rekognition's minimum confidence threshold to reduce API calls
- Monitor costs daily with AWS CloudWatch

### Risk 2: Poor Quality Images

**Impact:** High
**Probability:** High
**Mitigation:**

- Client-side image validation (resolution, size)
- Backend validation before Rekognition call
- Clear upload instructions for users
- Sample images shown in UI

### Risk 3: Manual Review Bottleneck

**Impact:** Medium
**Probability:** High
**Mitigation:**

- Prioritize high-confidence cases (88-94%)
- Auto-approve 95%+ confidence
- Auto-reject < 85% confidence
- Add more moderators if queue > 50 pending
