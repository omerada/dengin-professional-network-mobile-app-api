## 📊 Proje Özeti (Hızlı Bakış)

### Proje Temel Bilgileri

**Proje Adı:** Meslektaş  
**Tip:** Meslek Doğrulamalı Kapalı Sosyal Ağ  
**Platform:** React Native (iOS + Android)  
**Backend:** Spring Boot (Java)  
**Database:** PostgreSQL  
**Durum:** MVP Planlama Aşaması

### MVP Hedefleri

- 🎯 **Kullanıcı:** 500-2000 doğrulanmış kullanıcı
- 🎯 **Meslek Grupları:** 5 pilot meslek
- 🎯 **Süre:** 24 hafta (6 ay)
- 🎯 **Ekip:** 5-6 kişi
- 🎯 **Rating:** >4.0/5 (App Store/Play Store)

### Temel Özellikler

✅ AI destekli meslek doğrulama  
✅ Meslek bazlı kapalı feed sistemi  
✅ Gerçek zamanlı sohbet (grup + özel)  
✅ Gönderi paylaşma ve etkileşim  
✅ KVKK uyumlu veri yönetimi  
✅ Push notification  
✅ Profil yönetimi  
✅ Moderasyon ve bildiri sistemi

### Teknoloji Stack

**Frontend:**

- React Native 0.72+
- Expo SDK 49+
- Zustand / Redux Toolkit
- React Navigation
- Axios + React Query
- Socket.io Client

**Backend:**

- Spring Boot 3.x
- Spring Security + JWT
- Spring Data JPA + Hibernate
- Spring WebSocket
- PostgreSQL
- AWS S3 / Cloudinary

**DevOps:**

- GitHub Actions (CI/CD)
- AWS EC2 / DigitalOcean
- Firebase (Analytics, FCM)
- Sentry (Error tracking)

## 📝 Doküman Güncelleme Politikası

### Versiyon Kontrolü

Tüm dokümantasyon dosyaları Git ile versiyon kontrolü altındadır:

```
git commit -m "docs: [Doküman Adı] - [Yapılan Değişiklik]"
```

**Örnek:**

```
git commit -m "docs: API Specifications - Yeni mesajlaşma endpoint'leri eklendi"
```

### Güncelleme Sıklığı

- **Haftalık:** Sprint progress, blocker'lar
- **Sprint Sonunda:** Sprint retro sonuçları, güncellenmiş planlar
- **Milestone'larda:** İlgili teknik dokümantasyon
- **Gerektiğinde:** İş kuralları, API değişiklikleri

### Review Süreci

1. Doküman güncellemesi yapılır
2. Pull request oluşturulur
3. En az 1 kişi review yapar
4. Onay sonrası main branch'e merge
5. Ekip Slack'te bilgilendirilir
